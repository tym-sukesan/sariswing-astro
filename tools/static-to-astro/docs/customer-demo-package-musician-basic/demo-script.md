# Demo Script — Musician Basic Admin

**用途:** 顧客に管理画面イメージを見せるときの進行台本  
**前提:** [local-only preview route](../local-only-admin-preview-route.md) で画面を表示していること

---

## 1. 最初に伝えること

デモを始める前に、必ず次の点を口頭で伝えてください。

- **これは完成版ではなく、管理画面のイメージ（試作）です。**
- **今見せている画面では、実際のデータは保存されません。**
- **お客様の本番サイトには一切影響しません。**
- **今日の目的は、「どんな画面で、どんな項目を更新できるか」を一緒に確認することです。**
- **気になる点・足りない項目・不要な項目があれば、遠慮なくお知らせください。**

補足（必要なら）:

- 画面の色や文言は、正式導入時に調整できます。
- スマホからの更新も想定していますが、今日は主に PC 画面を見ていただきます。

---

## 2. デモの流れ

おおよそ **30〜45 分** を目安に、次の順番で進めます。

### 1. 管理画面の全体像

- 左側のメニュー（またはタブ）で、どの項目があるかを説明する。
- 「日常でよく触る項目」と「たまに触る項目」を分けて伝える。
- Dashboard（トップ）から始める。

### 2. Dashboard

- 直近の更新予定やお知らせの概要が見えるイメージであることを説明する。
- 「ログイン後、まずここを見る」という導線を伝える。

### 3. Profile

- 名前・プロフィール文・プロフィール写真を更新できることを説明する。
- 「About ページに載る内容」と結びつけて説明する。

### 4. Schedule / Live

- ライブ・レッスン・イベントの追加・編集ができることを説明する。
- 日付・会場・チケット情報など、よく使う項目を確認する。

### 5. Discography

- リリース情報（タイトル・アーティスト名・カバー画像）を管理できることを説明する。
- 作品数が少ない場合でも、追加しやすいかを確認する。

### 6. Links

- SNS・公式サイト・配信リンクなどをまとめて管理できることを説明する。
- リンクの追加・並び替えがしやすいかを確認する。

### 7. News

- お知らせ・活動報告を投稿できることを説明する。
- タイトル・本文・公開日のイメージを伝える。

### 8. Media

- 画像を管理する画面であることを説明する。
- **今日は実際のアップロードは行いません** と明記する。
- 画像の権利確認が必要になることにも軽く触れる。

### 9. Publish

- Staging（確認用）と Production（本番）の違いを説明する。
- **本番公開は必ず承認後** であることを伝える。
- 今日の画面では公開ボタンは動作しない（試作段階）ことを伝える。

### 10. 顧客からのフィードバック確認

- [customer-feedback-form.md](./customer-feedback-form.md) の項目を参考に、感想を聞く。
- 「分かりやすかった画面」「分かりにくかった画面」「追加したい項目」をメモする。

---

## 3. 顧客に確認すること

デモの途中・最後に、次の点を確認してください。

| 確認項目 | 聞き方の例 |
| --- | --- |
| 画面の分かりやすさ | 「メニューの名前は直感的でしたか？」 |
| 更新したい項目が揃っているか | 「普段更新したい内容は、この中にありそうですか？」 |
| 不要な項目があるか | 「使わなそうな項目はありますか？」 |
| 頻繁に更新する項目 | 「月に何回くらい、どの項目を更新しますか？」 |
| 画像管理の不安 | 「写真の入れ替えは、ご自身でできそうですか？」 |
| 公開フローの不安 | 「本番に出す前に、確認用サイトで見たいですか？」 |
| スマホで使いたいか | 「外出先からスマホで更新したい場面はありますか？」 |
| 複数人で管理するか | 「スタッフや事務の方も一緒に更新しますか？」 |

---

## 4. 最後に伝えること

デモの締めくくりとして、次を伝えてください。

- **次のステップは、今日のフィードバックをもとに、実データ接続前の設計確認です。**
- **正式導入時は、まず staging（確認用サイト）で安全に動作確認を行います。**
- **本番公開は、必ずお客様の承認後に行います。**
- **画像・テキストには権利確認が必要です。ご自身の素材か、使用許可があるものをお願いします。**
- **ご不明点があれば、いつでもお問い合わせください。**
