# Customer Admin Manual — Musician Basic

**対象テンプレート:** `musician-basic`  
**対象読者:** ミュージシャン本人、音楽教室・小規模スタジオ運営者、Webに詳しくない方  
**Phase:** G-5q — 顧客・運用者向け Admin Manual

このマニュアルは、CMS Kit 導入後に **NEWS / Schedule / Profile / Links / Media / Publish** を更新する方向けの説明です。  
画面の見た目やボタン名は、導入時の設定によって少し異なる場合があります。

> **開発者向け注記:** 現時点（G-5q）の管理画面は **scaffold（試作）段階** です。実際のログイン・保存・公開は、runtime 接続後に有効になります。`productionReady: false` / `connectedToRuntime: false`

---

## 1. この管理画面でできること

管理画面では、次の内容を **自分で更新** できます。

| 項目 | できること |
| --- | --- |
| **Profile / About** | 名前・プロフィール文・プロフィール画像を編集 |
| **Schedule / Live** | ライブ・イベントの追加・編集・並び替え |
| **Discography** | リリース情報（タイトル・アーティスト名・カバー画像）の編集 |
| **Links** | SNS・公式サイトなど外部リンクの追加・編集 |
| **News** | お知らせの追加・編集 |
| **Media（画像）** | イベント画像・カバー画像などの管理 |
| **Staging 公開** | 本番公開前の確認用サイトへ反映 |
| **Production 公開** | **承認制** — 本番サイトへの反映（慎重に行う） |

**覚えておくこと**

- 内容を編集して **保存** しただけでは、必ずしも本番サイトにすぐ出るわけではありません。
- まず **Staging（確認用）** で見た目を確認し、問題なければ **Production（本番）** 公開を検討します。
- Production 公開は **管理者（admin）** のみが行え、**顧客承認** が必要です。

## 2. ログインと権限

### ログイン

1. 管理画面の URL を開きます（導入時にお渡しします）。
2. メールアドレスとパスワードを入力してログインします。
3. ログインできない場合は、パスワード再設定を試すか、サイト管理者に連絡してください。

### 権限の種類

| 権限 | 主な役割 |
| --- | --- |
| **viewer（閲覧者）** | 内容を **見るだけ**。編集・公開はできません。 |
| **editor（編集者）** | Profile / Schedule / Discography / Links / News を **編集・保存** できます。Production 公開はできません。 |
| **admin（管理者）** | 編集に加え、**Media 管理** や **Staging / Production 公開** ができます。 |

### admin だけができること

- 画像ライブラリの管理
- Staging 公開の実行
- Production 公開の実行（承認ゲート通過後）
- 他ユーザーの権限に関する設定（導入設定による）

### editor ができること

- 各コンテンツの追加・編集・保存
- 公開 / 非公開の切り替え（項目ごと）
- 並び順の変更
- 複製・削除 / 復元（論理削除）

### viewer ができること

- Dashboard や各項目の **閲覧のみ**
- 編集ボタンは使えない、または保存できない

### パスワード再設定

- 「パスワードを忘れた」リンクから、登録メールアドレス宛に再設定メールが届きます。
- メールが届かない場合は、迷惑メールフォルダを確認してください。
- それでも解決しない場合は、サイト管理者（制作側）に連絡してください。

> **開発者向け注記:** ログイン・権限チェックは G-5m-b Auth scaffold をベースに、runtime 接続後に Supabase Auth 等と接続予定です。現時点では実装前です。

## 3. Dashboard

Dashboard は **サイト全体の状態** を一目で確認する画面です。

### 確認できること

- **未公開の件数** — 下書きのままの News や Schedule など
- **画像レビュー待ち** — 確認が必要な画像があるか
- **Staging / Production の状態** — 最後に公開した日時、進行中かどうか
- **公開前チェック** — 本番公開前に確認すべき項目の目安

### 使い方のコツ

- 更新作業の **最初と最後** に Dashboard を開く習慣をつけると安全です。
- 「レビュー待ち」や「未公開」が残っていないか確認してから Staging / Production 公開を検討してください。

## 4. Profile / About の更新

サイトの **顔** になる情報です。

### 編集できる項目

| 項目 | 説明 |
| --- | --- |
| **名前** | アーティスト名・活動名 |
| **プロフィール本文** | 自己紹介・経歴・活動内容 |
| **プロフィール画像** | 顔写真やアーティスト写真 |

### 手順（一般的な流れ）

1. **Profile** メニューを開く。
2. 名前・本文を入力または修正する。
3. 画像をアップロードする場合は、Media または Profile 画面から画像を選ぶ。
4. **保存** を押す。
5. **Staging** で表示を確認する。
6. 問題なければ、必要に応じて Production 公開を依頼する。

### 注意

- 保存前に **誤字・表記ゆれ** を読み返してください。
- プロフィール画像も **権利確認** が必要です（自分の写真、使用許可のある素材のみ）。
- 保存しただけでは本番に出ない場合があります。Staging 確認 → 公開の流れを守ってください。

## 5. Schedule / Live の更新

ライブ・イベント情報を管理します。

### 編集できる項目

| 項目 | 説明 |
| --- | --- |
| **日付** | 開催日 |
| **タイトル** | イベント名 |
| **会場** | 会場名 |
| **詳細** | 補足説明（任意） |
| **画像** | フライヤー画像・トップ用画像 |
| **公開 / 非公開** | サイトに表示するかどうか |
| **並び順** | 一覧での表示順 |
| **複製** | 似たイベントをコピーして編集 |
| **削除 / 復元** | 削除は論理削除（復元可能） |

### 画像について（重要）

Schedule には **2種類の画像** がある場合があります。

| 種類 | 用途 |
| --- | --- |
| **flyer（フライヤー）** | イベント詳細ページなどで使う画像 |
| **home（トップ用）** | トップページや一覧で使うサムネイル |

**注意点**

- **日付と画像の内容が一致しているか** 確認してください（別の日のフライヤーを付けない）。
- 画像がない場合は **NO PHOTO / placeholder（仮画像）** が表示されることがあります。意図的でない場合は画像を追加してください。
- フライヤーやトップ用画像は、**human review（人による確認）** が必要な場合があります。アップロード後、Dashboard で「レビュー待ち」になっていないか確認してください。

### 公開 / 非公開

- **公開 ON** — Staging / 本番（公開済みの場合）に表示対象になります。
- **公開 OFF** — サイトには表示されません（下書き状態）。

## 6. Discography の更新

アルバム・シングルなどの **リリース情報** を管理します。

### 編集できる項目

| 項目 | 説明 |
| --- | --- |
| **タイトル** | アルバム名・曲名 |
| **アーティスト名** | 表記上のアーティスト名 |
| **カバー画像** | ジャケット画像 |
| **公開 / 非公開** | 一覧に載せるかどうか |
| **並び順** | 表示順（新しい順など） |

### 手順

1. **Discography** を開く。
2. 新規追加または既存項目を編集。
3. カバー画像をアップロード（JPEG / PNG / WebP 推奨）。
4. 公開設定と並び順を確認。
5. 保存 → Staging で表示確認。

### 注意

- ジャケット画像は **レーベル・デザイナーの使用許可** があることを確認してください。
- カバーが未設定の場合、プレースホルダー表示になることがあります。

---

## 7. Links の更新

SNS・ストリーミング・公式サイトなど **外部リンク** を管理します。

### 編集できる項目

| 項目 | 説明 |
| --- | --- |
| **ラベル** | ボタンや一覧に表示する名前（例: Instagram、Spotify） |
| **URL** | リンク先の Web アドレス |
| **並び順** | 表示順 |
| **公開 / 非公開** | 表示するかどうか |

### 長い URL の扱い

- URL は **https://** から始まる完全なアドレスを入力してください。
- 長い URL でも問題ありませんが、**コピー＆ペースト** で入力し、前後に余分な空白が入らないように注意してください。
- 短縮 URL（bit.ly 等）も使えますが、**リンク先が正しいか** Staging で必ず確認してください。

---

## 8. News の更新

お知らせ・ニュースを管理します。

### 編集できる項目

| 項目 | 説明 |
| --- | --- |
| **タイトル** | 見出し |
| **日付** | 掲載日 |
| **本文** | お知らせの内容 |
| **外部 URL** | 詳細を別ページに誘導する場合（任意） |
| **公開 / 非公開** | 表示するかどうか |
| **削除 / 復元** | 論理削除（復元可能） |

### 手順

1. **News** を開く。
2. 「新規追加」または既存記事を編集。
3. タイトル・日付・本文を入力。
4. 公開 ON/OFF を設定。
5. 保存 → Staging で表示・リンクを確認。

### 注意

- 日付が **未来** になっていないか確認してください（意図的でない限り、掲載日は今日または過去が一般的です）。
- 外部 URL を設定した場合、リンク先が正しく開くか確認してください。

## 9. 画像管理

### 使用できる画像形式

- **JPEG**
- **PNG**
- **WebP**

（導入設定により異なる場合があります）

### サイズの目安

- 長辺 **1600px 程度** を目安にしてください。
- ファイルが大きすぎると、アップロードや表示が遅くなることがあります。
- スマホ向けサイトでは、大きすぎる画像は自動で縮小される場合があります。

### 画像の権利確認（重要）

次のいずれかに当てはまる画像だけを使ってください。

- 自分で撮影・作成したもの
- 使用許可を得た素材
- レーベル・会場・デザイナーから **使用 OK** をもらったもの

**staging-only（確認用のみ）**

- 権利確認が完了する前の画像は、**Staging のみ** で確認し、Production には出さない運用にしてください。

**production 前の owner confirmation（権利者確認）**

- 本番公開前に、「この画像を公開してよい」という **権利者の確認** が必要な場合があります。
- Dashboard や Media 画面で「権利確認待ち」が表示されたら、確認完了まで Production 公開を行わないでください。

### 差し替え時の注意

- 画像を差し替えたら、**Staging で表示を確認** してから本番反映を検討してください。
- 古い画像がキャッシュで残って見えることがあります。スマホ・PC 両方で確認してください。

## 10. Staging で確認する

### Staging とは

**Staging（ステージング）** は、本番公開 **前** に内容を確認するためのサイトです。

- 一般の検索エンジンには載りにくい設定（**noindex** / **robots Disallow**）になっていることが多いです。
- 制作側・お客様だけが確認できる URL です。
- ここで問題なければ、Production 公開を検討します。

### 確認手順

1. 管理画面で内容を保存。
2. **Staging 公開** を実行（admin 権限）。
3. Staging の URL をブラウザで開く。
4. 次を確認する:
   - **スマホ表示** — 実際の利用者はスマホが多いです
   - **画像** — 切れていないか、日付と合っているか
   - **リンク** — 外部 URL が正しく開くか
   - **誤字** — タイトル・本文・会場名

## 11. 本番公開

**Production 公開は慎重に行ってください。** 一般向けサイトに反映されます。

### 本番公開の前に

| 確認項目 | 説明 |
| --- | --- |
| **顧客承認** | 公開内容について、お客様（または責任者）の OK を得る |
| **権利確認** | 画像・テキストの使用許可が取れている |
| **rollback plan** | 問題があった場合の戻し方（制作側と事前に確認） |
| **admin 権限** | Production 公開は admin のみ |
| **production publish enabled** | サイト設定で本番公開が有効になっているか（制作側が管理） |

### 誤って押さない設計

- Production 公開ボタンは、**確認ダイアログ** や **承認ゲート** を通過する設計になっています。
- 「Staging 公開」と「Production 公開」は **別ボタン** です。混同しないでください。
- 不安な場合は、**Staging 公開だけ** 行い、Production は制作側に依頼してください。

## 12. よくある質問

### 更新したのに本番サイトに出ないのはなぜ？

- **保存** したか確認してください。
- 項目の **公開 ON** になっているか確認してください。
- 保存だけでは本番に出ない場合があります。**Staging 公開 → Production 公開** の流れが必要です。
- Production 公開は admin が実行し、承認が必要です。

### Staging と本番の違いは？

| | Staging | Production（本番） |
| --- | --- | --- |
| 目的 | 公開前の確認 | 一般向けの本番サイト |
| 検索エンジン | 載りにくい（noindex 等） | 公開される |
| 誰が見る | 確認用 | 一般の訪問者 |
| 公開操作 | Staging 公開 | Production 公開（承認制） |

### 画像が表示されない時は？

- アップロードが完了しているか確認。
- ファイル形式（JPEG / PNG / WebP）が対応しているか確認。
- ファイルサイズが大きすぎないか確認。
- Staging で表示されているか確認（本番だけ未反映の可能性）。
- 権利確認待ちでブロックされていないか Dashboard を確認。

### 非公開にしたらどうなる？

- サイトの一覧・詳細から **表示されなくなります**。
- データは削除されません。再度 **公開 ON** にすれば表示できます。

### 削除と復元の違いは？

- **削除** — 論理削除。一覧から消えますが、**復元** できます。
- **復元** — 削除した項目を再度使える状態に戻します。
- 完全に消えて戻せない削除は、通常は管理者のみが行います。

### URL が長い場合は？

- 長い URL でも入力できます。コピー＆ペーストで入力し、Staging でリンクをクリックして確認してください。

### スマホで確認した方がいい？

- **はい。** 多くの訪問者はスマホで見ます。Staging 公開後、スマホのブラウザで必ず確認してください。

### 間違えて公開した場合は？

- すぐに **非公開** に切り替えられる項目は、公開 OFF にしてください。
- Production 公開後の修正は、Staging で直してから再度公開する、または制作側に **rollback（巻き戻し）** を依頼してください。
- 慌てず、Dashboard の状態と制作側に連絡してください。

### パスワードを忘れた場合は？

- ログイン画面の「パスワードを忘れた」から再設定メールを送ってください。
- 届かない場合は迷惑メールを確認し、それでもダメなら制作側に連絡してください。

### 画像の権利確認とは？

- その画像を **Web 上で公開してよいか** を確認することです。
- 他人の写真・デザイン・ロゴを無断で使わないでください。
- 確認が取れるまで Staging のみで確認し、Production には出さないでください。

## 13. トラブル時の連絡前チェック

制作側やサポートに連絡する前に、次を確認してください。

- [ ] **保存** したか
- [ ] **公開 ON** になっているか
- [ ] **Staging** で確認したか
- [ ] 画像が **大きすぎない** か
- [ ] **URL** が間違っていないか（https://、余分な空白）
- [ ] **日付** が意図どおりか（未来 / 過去の誤り）
- [ ] 今は **本番反映前** か、本番反映後か

これらを確認したうえで連絡すると、解決が早くなります。
