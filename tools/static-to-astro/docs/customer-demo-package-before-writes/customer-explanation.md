# お客様向け説明（CMS Kit staging デモ）

対象: Wix / STUDIO / Jimdo などからの乗り換えを検討されている、ミュージシャン・音楽教室・小規模事業者の方

---

## この CMS Kit で目指していること

- **自分のサイトを、長く使える形で持ちたい** — デザインと中身を、事業の成長に合わせて更新できる
- **更新のしやすさと安全性の両立** — 誰でも触れるが、誤って壊したり公開してしまうリスクを減らす
- **段階的な導入** — いきなり本番を変えるのではなく、確認しながら進める

## Wix などのサービスとの違い（ざっくり）

| 観点 | 一般的なサイトビルダー | いまお見せしている方向性 |
| --- | --- | --- |
| 始めやすさ | すぐ編集・公開できる | 最初は **確認用画面** から。本番は準備が整ってから |
| データの持ち方 | サービス内に閉じることが多い | 将来、**ご自身のデータベース** と連携する設計を検討 |
| カスタム | テンプレート中心 | ミュージシャン向けの **スケジュール・作品・リンク** などに特化しやすい |
| リスク管理 | 公開ボタンがすぐ使える | **保存・公開は安全確認後** に段階的に有効化 |

※ 製品の最終形態・料金は別途ご案内します。本資料は **現時点のデモ** の説明です。

## 現時点でできること

- 管理画面の **見た目・レイアウト** の確認
- **ログイン画面** のイメージ確認
- **プロフィール・スケジュール・作品・リンク・お知らせ** の **表示確認**（読み取りのみ）
- 「誰がログインしているか」「書き込みがまだ無効か」の **状態表示**

## まだできないこと（意図的に無効）

- **保存・新規作成・更新・削除**
- **画像のアップロード**
- **サイトの公開・更新**（本番への反映）
- 本番の `/admin` 管理画面への接続

**本番サイトには影響しません。** データの削除や上書きも行いません。

## なぜ書き込み機能を急がないのか

- 誤って **下書きが公開** されたり、**データが消えた** りする事故を防ぐため
- お客様ごとに必要な項目（例: レッスン日程の項目数）が違うため、**先に表示と要望を揃える**
- データベースの **アクセス権（RLS）** を、読み取りのあとに書き込み用として設計するため

「未完成」ではなく、**安全確認しながら段階的に進めている** 段階です。誇張せず、現状を正直にお伝えします。

## 今後の流れ（イメージ）

```txt
① 今日のような staging デモで見た目・表示を確認
② ご要望の整理（何を・誰が・どの頻度で更新したいか）
③ 保存・公開の安全設計
④ staging で保存・公開の試験
⑤ 本番導入のご判断
```

## 安心していただきたいこと

- 今お見せしているのは **デモ / staging 確認** です
- **保存・公開はまだ無効** です
- **本番サイトには影響しません**
- **データの削除や上書きは行いません**

ご不明点はデモ後の [feedback-form.md](./feedback-form.md) または担当者までお知らせください。
