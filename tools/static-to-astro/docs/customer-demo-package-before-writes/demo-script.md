# Demo Script（顧客デモ台本）

**対象 URL（ローカル）:** `http://localhost:4321/__admin-staging-shell/musician-basic/`  
（staging preview URL がある場合は当日差し替え可 — commit しない）

**所要時間の目安:** 25〜40 分（質疑含む）

---

## 0. オープニング（2分）

「本日は、ホームページ更新の仕組みを **安全に段階的に作っている途中** の画面をお見せします。  
**本番の管理画面や本番サイトそのものではありません。** 見た目・ログイン・データの**表示確認**ができる段階です。」

---

## 1. staging shell であること（3分）

**見せるもの:** ページ上部の説明・ルートパス

**話すこと:**

- これは **staging shell**（試作・確認用）です
- 本番の `/admin/` 管理画面には **まだ接続していません**
- 本番サイトの表示やデータを **書き換えることはありません**
- 検索エンジンに載せない設定（noindex）の開発用ルートです

---

## 2. ログイン UI（5分）

**見せるもの:** ログインセクション

**話すこと:**

- 将来、管理者だけが入れる **ログイン画面** のイメージです
- 現在は **staging 用の Supabase Auth** または **モック（仮）表示** で確認できます
- 本番の顧客アカウントとは別の、**確認専用の環境** です

---

## 3. Auth status / role status（4分）

**見せるもの:** Auth ステータスパネル、role / allowlist 表示（該当時）

**話すこと:**

- 「誰がログインしているか」「どの権限か」を画面に示す想定です
- いま表示されているロールは **確認用** で、本番の厳密な権限管理はこれから設計します
- 複数人で更新する場合は、後から **権限の分け方** を一緒に決められます

---

## 4. read-only data preview（8分）

**見せるもの:** Read-only data adapter status パネル

**話すこと:**

> 今の段階では、管理画面の見た目・ログイン導線・データ表示の確認ができます。  
> 保存や公開などの書き込み操作は、安全確認が終わるまで無効にしています。

- **Provider:** mock（仮データ）または supabase read-only（staging の読み取りのみ）
- **`canWrite: false`** — 書き込み不可であることを明示
- **Storage / Publish:** 未接続・無効
- **productionReady: false** — 本番提供前の段階

---

## 5. 各モジュールの説明（8分）

| モジュール | 説明のポイント |
| --- | --- |
| **profile** | 名前・紹介文・画像など、トップに出す基本情報 |
| **schedule** | ライブ・レッスン日程。公開済みのみ表示する想定 |
| **discography** | 作品・リリース情報 |
| **links** | SNS・配信など外部リンク |
| **news** | お知らせ。テーブルが無い staging では **この部分だけエラー表示** になり得るが、**画面全体は壊れない** |

**補足:** モック表示では `example.com` の仮 URL のみ使用。実顧客 URL はデモ資料に含めない。

---

## 6. 書き込み・保存・公開が無効であること（3分）

**はっきり伝える:**

- 保存・新規作成・更新・削除ボタンは **意図的に無効**
- 画像アップロード・サイト公開・FTP デプロイも **まだ行わない**
- データの削除や上書きの心配は **不要**

---

## 7. 今後の流れ（4分）

```txt
staging で見た目・表示を確認
    ↓
安全設計（RLS・書き込み計画）
    ↓
staging で保存・公開の試験（承認ゲート付き）
    ↓
本番導入の判断
```

- 次の技術フェーズは **G-6-a: write operation safety plan**（書き込みの安全設計）から始める
- 急いで書き込みを有効にせず、**誤公開・誤削除を防ぐ** 順番を取る

---

## 8. 顧客に確認したいこと（5分）

[feedback-form.md](./feedback-form.md) の質問を口頭またはフォームで回収。

**クロージング:**

「本日見ていただいたのは **確認用の staging** です。ご要望をいただいたうえで、次に **保存・公開を含めるか・どの項目からか** を一緒に決めていきます。」
